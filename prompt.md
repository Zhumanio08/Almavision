Ты — senior fullstack developer. Необходимо разработать полностью рабочий MVP веб-дашборда "Almavision" для управления городом на чистом HTML/CSS/JS (БЕЗ React, БЕЗ Vite, БЕЗ сборщиков). Проект должен открываться напрямую через index.html и работать без сервера (или с простым HTTP-сервером для роутинга).

## Данные: 8 районов Алматы

Районы: 
- Наурызбайский
- Алмалинский
- Алатауский
- Турксибский
- Бостандыкский
- Медеуский
- Ауэзовский
- Жетысуский

### Транспортные показатели (норма / жёлтый / красный):
- Загруженность (%): норма 35-65, жёлтый 65-75, красный >75
- Скорость потока (км/ч): норма 35-60, красный <25
- Количество машин (тыс): норма 5-10, красный >12
- Пробочный индекс (1-10): норма 2-5, красный >7
- Общественная нагрузка (%): норма 50-75, красный >85

### Экологические показатели:
- AQI: норма 30-60, жёлтый 60-85, красный >85
- PM2.5 (мкг/м³): норма 10-25, красный >35
- PM10 (мкг/м³): норма 20-40, красный >50
- NO2 (мкг/м³): норма 10-30, красный >40
- CO2 (ppm): норма 380-450, красный >500
- Шум (дБ): норма 50-60, красный >70

### Временные срезы (для графиков):
Часы: 08:00, 10:00, 12:00, 14:00, 16:00, 18:00, 20:00, 22:00, 00:00
Для каждого района сгенерируй правдоподобные данные с утренним и вечерним пиком.

## Функциональные требования

### 1. Главная страница (Dashboard):
- Карточки всех районов с KPI (загруженность, AQI, приоритет). Цвет карточки = статус по худшему показателю.
- Сводная таблица "Рейтинг районов" по приоритету вмешательства.
- Кнопка "Обновить данные" — генерирует новые случайные данные для всех районов и временных рядов.
- Место для AI-ассистента (заглушка с текстом "Здесь будет ChatGPT-ассистент").

### 2. Страница каждого района (роутинг через hash или простой JS-роутер):
- Название района.
- Два графика (Line Chart): транспорт (загруженность + скорость) и экология (AQI + PM2.5). Использовать Chart.js CDN.
- KPI-блок со всеми показателями (транспорт + экология). Красный/жёлтый/зелёный индикатор у каждого.
- AI-панель с анализом:
  * Что происходит (отклонения от нормы)
  * Насколько критично (высокий/средний/низкий приоритет)
  * Рекомендации (конкретные действия)
  * Прогноз на 2 часа
- Кнопка "Вернуться к списку районов".

### 3. AI-логика (на чистом JS):
- Детекция аномалий по порогам (красный/жёлтый).
- Приоритет: высокий если есть красный, средний если только жёлтые, низкий если все зелёные.
- Если и транспорт, и экология красные — комплексная рекомендация.
- Прогноз: линейная экстраполяция по последним двум точкам временного ряда.

### 4. Технологии и стиль:
- Чистый HTML/CSS/JS (без React, без сборщиков)
- Chart.js (CDN) для графиков
- Material UI можно использовать через CDN (MUI Core) или просто написать свои компоненты в стиле MUI
- Цветовая схема: зелёный (#2e7d32), красный (#d32f2f), белый фон.
- Логотип справа сверху, кликабельный (ведёт на главную).
- Адаптивная вёрстка (Flex/Grid).

- AI-текст на русском, управленческим стилем.
- Роутинг должен работать без сервера (через hash или History API с fallback).
- Код должен быть чистым, без комментариев комментариями.
